# Support

Use this document to choose the right support channel.

## What to Use

- Bug reports: GitHub Issues (Bug Report template)
- Feature requests: GitHub Issues (Feature Request template)
- Security vulnerabilities: Follow `SECURITY.md` and use private reporting
- Questions and usage help: GitHub Discussions

## Before Opening an Issue

Please include:

- OS and version
- Docker version
- How Docker is connected (`socket`, `ssh`, or `tcp`)
- Steps to reproduce
- Expected behavior and actual behavior
- Relevant logs and screenshots

## Scope of Support

Maintainers will try to help with:

- Reproducible bugs in supported versions
- Setup and upgrade guidance
- Security triage and mitigation guidance

Maintainers may not be able to provide:

- One-off custom development
- Guaranteed response SLAs
- Support for heavily modified forks

## Response Expectations

- Community responses may be asynchronous.
- Maintainer response time depends on availability.
- Critical security reports are prioritized when submitted through private channels.
